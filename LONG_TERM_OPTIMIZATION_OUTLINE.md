# 山巅微光长期优化总纲

> 记录时间：2026-05-31  
> 目标：做一个原创、可信、可持续扩展的精密平台跳跃游戏。参照优秀作品的产品经验，但不复刻关卡、角色、美术、叙事或具体表现。我们的路线不是复刻，而是用更强练习系统、更清晰反馈和更稳定工程质量形成自己的优势。

## 1. 产品北极星

山巅微光要成为一个“越练越清楚、越快越好看、越难越公平”的平台跳跃游戏。我们的优势不该只是难度，而是让玩家每一次失败都更接近成功：输入可信、路线可读、复盘有用、重开迅速、高手路线有观赏性。

最终体验应该同时满足三类玩家：

- 新玩家：能看懂为什么死，知道下一步该练什么，不会被菜单和等待打断。
- 进阶玩家：能用房间训练、Focus、ghost、split 和路线提示稳定提升。
- 高手玩家：能追求极限路线、低死亡、S 级、Flow、无失误连段和更高风险的表达。

## 2. 已有基础

当前版本已经有一条可以继续扩张的骨架：

- 移动：跳跃、冲刺、抓墙、墙跳、快速下落、冲刺后续跃窗口、光继点、风升流、超载棱镜、回声锚点、脆冰平台。
- 练习：房间选择、当前房间重开、split delta、PB/SPLIT 弹窗、房间 target、最佳路线 ghost、当前路线轨迹、死亡轨迹回放。
- 诊断：死亡原因、Focus 房间画像、房间技能标签、练习教练、Focus 跳转与重置。
- 发布：GitHub Pages 公开访问，`npm run check` 覆盖地图、HTML 双文件、练习 UI、部署前质量门。

这说明下一阶段不用推倒重来。我们要做的是把这些零件整理成一个更强的“训练型平台跳跃系统”。

## 3. 长期推进主线

### A. 手感可信

优先级最高。任何系统都不能补救糟糕手感。

- 输入响应：继续检查跳、冲、抓、快速下落、斜向冲刺、转向、落地后的即时动作。
- 宽限策略：跳跃缓冲、土狼时间、墙面宽限、角落修正要稳定，但不能让玩家误以为自己已经站稳。
- 重生状态：死亡、重开、跳房间后必须清空残留输入和移动状态。
- 高阶动作：Spark hop、墙跳节奏、棱镜加速、风中快速下落要成为我们自己的动作语言。
- 手柄/触控：不能只是“能按”，要有稳定方向、清晰 UI 和合理误触保护。

### B. 关卡可读

难度要来自路线执行，而不是看不懂。

- 每个房间保留三层路线：安全通关线、流畅进阶线、高手高风险线。
- 每个新机制先安全出现，再进入压力组合，最后才进入终盘考试。
- 前 10 房继续作为垂直切片，不急着堆数量；先把教学、组合、压力、终局四段做顺。
- 后续章节可以围绕不同主题扩展：风、棱镜、回声、脆冰、光继点连锁、墙面节奏。
- 每个房间都要有技能标签、训练目的、target、S 线逻辑和失败原因预期。

### C. 练习系统领先

这是我们最有机会超出同类体验的方向。

- 房间 Drill：一键开始推荐房间训练，目标可以是无失误、低死亡、低于 target、减少某个死亡原因。
- 路线复盘：通关后显示最拖慢的房间、最常见死亡、最该练的下一房、当前 clean clear 数。
- 个人教练：Focus 不只标记薄弱点，还要给“下一步目标”，避免玩家不知道怎么练。
- Ghost 分层：最佳 ghost、当前 ghost、target ghost 或示范线要可开关，不能干扰危险物读取。
- 长期档案：记录每个房间 PB、S 级、clean、死亡类型、练习次数、最近进步。

### D. 美术与反馈克制而有辨识度

美观不是堆特效，而是让动作更好读、让世界更有气质。

- 主角：小人要有清楚轮廓、方向感、动作姿态和颜色锚点，避免“临时方块感”。
- 世界：山巅、冰、夜光、风、棱镜、回声可以形成冷亮但不单调的视觉语言。
- 特效：冲刺、落地、擦边、成功连段、棱镜超载、脆冰碎裂都要服务状态表达。
- 震动：默认轻，允许关闭；失败震动不能让玩家不舒服。
- 可读性：危险物、可站立物、背景、粒子、ghost 必须分层清楚。

### E. 工程和发布可持续

大推进必须能长期活下去。

- 所有房间数据必须经过检查脚本，防止难度曲线塌掉。
- 所有练习 UI 的关键 DOM、设置、热键隔离、触控方向都要有契约检查。
- 每次公开更新都跑本地检查、本地浏览器 smoke、git diff、commit、push、GitHub Pages workflow、线上校验。
- 文档要随功能更新，尤其是操作、训练系统、路线规则和质量门。
- 尽量先把一个系统做薄但完整，再逐步加深，避免堆半成品。

## 4. 审美与可读性原则

- 画面第一眼要知道：我在哪、我是谁、哪里危险、下一步大概往哪走。
- UI 保持工具感，少解释、多反馈；训练信息靠状态、标签、弹窗和房间目录承载。
- 粒子不能像实体平台，背景不能抢危险物，ghost 不能遮挡路线。
- 色彩不要只靠一种蓝紫或一种冷色；主角、危险、可互动、背景要有明确层级。
- 角色动画先追求姿态清楚，再追求细节华丽。
- “漂亮”必须和“可操作”绑定，不能为了截图牺牲读秒级反应。

## 5. 分阶段路线

### 阶段 1：10 房垂直切片打磨

目标：让当前公开版本变成一个完成度很高的小作品。

- 调整手感边界：Spark hop、墙跳、冲刺方向、风中下落、棱镜速度。
- 加房间 Drill 和通关复盘。
- 补房间 medal/clean/S 级状态。
- 重做或强化主角形象和关键动作反馈。
- 再审第 1-10 房节奏，减少突兀难点，让终盘难但公平。

### 阶段 2：第一章节成型

目标：扩到 20-30 个房间，但保持质量。

- 按机制主题分组，而不是随机加房间。
- 增加小型章节结构、章节选择、章节 PB、章节 clean。
- 让每个主题都有安全线、进阶线、高手线。
- 引入少量叙事和环境变化，但不压过操作。

### 阶段 3：长期练习平台

目标：让它不只是一次性通关，而是可反复挑战。

- 每日/每周挑战房。
- 进阶训练课表。
- 本地 replay/ghost 管理。
- 房间编辑器或简化地图导入格式。
- 社区分享路线和挑战码。

### 阶段 4：公开测试与版本化

目标：让外部玩家持续进入、反馈、复玩。

- 发布稳定 demo 页面。
- 收集操作、难度、卡点、设备兼容反馈。
- 建立 known issues 和版本 changelog。
- 根据数据重排新手路线和训练建议。

## 6. 下一轮推进包

近期最值得做的一组任务：

1. 房间 Drill：从练习教练一键进入推荐房间，目标先做“无失误通过”和“少死一次”。
2. 登顶复盘增强：显示最弱房间、最大 split 损失、clean 数、下一练习建议。
3. 房间 medal：房间目录显示 S/A/B、clean、Focus 标记，让练习目标一眼可见。
4. 角色视觉 pass：替换临时形象，强化站立、奔跑、跳跃、冲刺、攀墙状态。
5. 震动舒适 pass：默认降低强度，失败震动更短，设置里保留完全关闭。
6. 特效清晰 pass：冲刺拖尾、落地尘、脆冰碎裂、棱镜超载都围绕可读性重做。
7. 第 1-10 房复查：每房写一句训练目的，检查安全线/进阶线/高手线是否都存在。
8. 线上 smoke 固化：把关键浏览器检查变成可重复脚本或半自动清单。

## 7. 完成标准

每次大推进至少满足：

- 本地 `npm run check` 通过。
- 本地浏览器 smoke：页面能打开、角色可动、设置不吞错输入、重开不残留状态、关键 UI 出现。
- 地图曲线检查：早期教学、中期组合、后期压力不能倒挂。
- 文档更新：README/ROADMAP/本总纲中相关部分不失真。
- GitHub Pages 部署成功，公开网址能看到新版本。

## 8. 风险边界

- 不复刻已有作品的关卡、角色、美术、故事或独有表达；我们学习的是产品质量和设计原则。
- 不为了“更难”牺牲公平性。难点必须可读、可练、可复盘。
- 不为了“更炫”牺牲危险物和角色状态的可读性。
- 不让练习辅助替玩家通关；辅助只减少无效重复，不隐藏真实计时和真实错误。
- 不一口气扩太多房间。宁愿少而精，先让垂直切片成立。
