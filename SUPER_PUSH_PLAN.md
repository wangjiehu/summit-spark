# 山巅微光超级推进计划

记录时间：2026-05-31

## 总判断

现在的目标不是“加几个功能”，而是把 `山巅微光` 推成一个可信的精密平台跳跃 demo。要超越参考作品，不能靠复刻，也不能只靠更难。真正的突破口是：

- 手感可信：玩家相信输入、碰撞、宽限和重开。
- 关卡可读：每个房间能看懂安全线、进阶线和高手线。
- 练习领先：死亡、split、ghost、Focus、Drill、medal 和复盘组成完整训练闭环。
- 原创动作语言：Spark hop、光继点、风、棱镜、回声和脆冰形成自己的节奏。
- 工程稳态：每次推进都能被脚本、浏览器 smoke 和文档约束住。

## 反思机制

每一批推进都必须先问四件事，再继续：

1. 这个改动是在提高“可练”和“可读”，还是只是在堆 UI？
2. 这个改动会不会让危险物、角色状态、ghost 或练习线互相遮挡？
3. 这个改动会不会让玩家误会计时、clean、PB 或 Focus 数据？
4. 这个改动能不能被质量门或浏览器 smoke 检查到？

如果答案不稳，就先缩小范围，改成更可验证的一步。

## 当前超级推进包

后续大步推进必须同时参考 `DEVELOPMENT_DIRECTION.md`，用它约束 PR、P2 数据来源、UI 优先级和质量门，避免训练系统、长期挑战和发布流程各自漂移。

### A. 练习入口

- Coach 推荐房间不只是跳转，而是进入 Drill。
- Drill 必须显示目标，而不是让玩家猜为什么被推荐。
- Drill 完成后反馈 clean/clear，并保留真实错误记录。
- 主 Drill 入口必须解析为 Clean/Pace/Style/Expert 真实合同，避免最显眼的训练按钮变成宽松通关。已完成基础版。
- 设置面板提供 Clean/Pace/Style/Expert 四种 Drill 变体，分别对应无失误、target、难度类型和高手线。已完成基础版。
- Style Drill 必须让每房难度风味不同：精度、恢复、节拍、连锁、路线、地形风险、读图和终盘耐压都要能被练到。已完成基础版。
- Style 目标必须进到游玩时反馈：入场卡、Drill HUD 和登顶复盘都要能读到类型目标。已完成基础版。
- Style/Expert 缺失动作必须在房间里有信标提示，让玩家知道该去找哪个机制。已完成基础版。
- Expert Drill 必须证明本房高手动作，不允许只用快速 safe 线混过。已完成基础版。
- Drill 未达标时应直接重开同一房间，减少菜单往返和错误练习延续。已完成基础版。
- 复盘区提供 Clean/Pace/Style/Expert 四张训练卡，直接从数据生成下一步，并显示合约尝试/完成进度。已完成基础版。
- 复盘区提供全房间掌握表，按训练优先级排序并一键进入对应 Drill。已完成基础版。

### B. 房间掌握度

- 房间目录显示 medal、PB/target、clean、技能标签和 Focus 标记。
- 入场卡显示 medal/clean，让玩家一进房就知道本房练习状态。
- 后续继续补 room purpose、三层路线和目标提示。

### C. 复盘系统

- 登顶复盘显示 clean 房间、Flow、死亡结构、最大 split 损失、最弱房和下一 Drill。
- 登顶复盘拆成卡片，最大 split loss 和薄弱原因都能落到具体路线建议。已完成基础版。
- 登顶复盘可直接进入下一 Drill 或最慢房 Drill，减少菜单往返。已完成基础版。
- 后续继续补每房 split loss 表和低摩擦重练入口。

### D. 视觉可读性

- 练习线和 ghost 必须可开关、可调透明度。
- 角色新增跳跃、冲刺、Spark、墙跳、光继、棱镜、弹簧、召回、出生、落地和死亡的状态脉冲。已完成基础版。
- 脆冰碎裂进度、风场边界、光继/棱镜冷却环和回声召回牵引线已完成基础版。
- 后续继续重做更完整的角色动画帧和章节级环境变化。

### E. 质量门

- 所有新 UI id 必须进 contract check。
- 所有新房间数据必须长度对齐 maps。
- 每轮结束跑 `npm run check`。
- 前端变更必须做本地浏览器 smoke。
- 中文练习文案要用中英文分流的契约检查，避免为了通过英文长度阈值而写废话。

### F. 设计可读性

- 计时只记录真实游玩，不记录出生后犹豫和设置阅读时间。已完成基础版。
- 设置面板打开时暂停模拟，降低误操作和数据污染。已完成基础版。
- 设置面板按画面、练习、复盘分组，练习文案优先中文化。已完成基础版。
- 窄屏 HUD 收紧信息密度，触控控件只在触控设备显示。已完成基础版。

## 自主推进节奏

后续尽量不问用户，除非遇到以下情况：

- 需要删除或重写用户未要求删除的内容。
- 需要访问外部账号、发布、推送或执行不可逆操作。
- 出现两种都合理但产品方向相冲突的选择。

默认推进顺序：

1. 先增强训练闭环。
2. 再补房间目的和复盘解释。
3. 再做手感与地图微调。
4. 再做角色/特效视觉 pass。
5. 最后扩房间数量。

## 2026-06-01 严格 UX 审查补充

这轮审查结论：训练系统已经足够厚，但小窗口的第一眼读图还不够稳。后续所有新增信息必须先过“跑动中是否能读懂”的门槛，再考虑展示更多数据。

- 窄屏 HUD 必须保持单行和低遮挡，优先保留房间号、pace 差值、死亡数和三条状态条；重复计时和收集数可以在小窗口隐藏。
- 入场目标必须让开 HUD，并在小窗口加大字号和提高底层对比度；房间目的、Style 目标和技能标签要能被一眼读到。
- 视觉优化必须服务读图：目标文本、危险线、pace 条、Flow 氛围和练习信标不能互相抢焦点。
- 下一批体验方向：死亡后的即时复盘更短、更靠近死亡点；Drill 失败提示要更像动作建议；设置面板继续减少“表格感”，把下一步训练动作放在最前。
- 已推进：设置面板顶部增加“下一步”训练主卡；死亡/手动重开后显示单一死亡教练提示，并压掉重复 Focus 弹层，避免玩家同时读两份相同建议。

## 2026-06-02 全面体验审查补充

这轮重点不是加更多系统，而是把已经很厚的练习系统变得更可靠、更少误操作、更适合小屏持续练习。

- 设置按钮和 HUD 统计从 `aria-hidden` 中解放出来，并同步 `aria-expanded` / `aria-controls`；只有装饰性状态条对辅助技术隐藏。
- 设置打开时给画面增加暂停态压暗，避免玩家误以为角色还在真实运行。
- Focus/Drill 统计清空改为带过期时间的二次确认，Reset 第一次只进入 armed 状态，第二次才执行。
- 出生/重开后的计时保护增加画面提示，玩家能看见“松开待命 / 首次输入开始计时”的状态。
- 练习队列卡增加“开练”动作 affordance；窄屏下掌握表隐藏次要统计，优先保留房间、建议动作和一键入口。
- 质量门补充设置可访问性、计时提示、Reset 确认、暂停视觉态和训练卡动作提示，防止后续回退。

## 2026-06-02 训练计划超大步

目标是把“练什么”从复盘表格升级成连续训练流，让玩家少读菜单，多进入下一次有效尝试。

- 设置面板新增三步训练计划：现在修最短板，接着换一种能力，然后补路线链；每一步都是可点击 Drill。
- 死亡教练从原因提示升级为动作处方：下一次该怎么按、该走哪条线、推荐进哪个 Drill 会同时出现。
- Drill 失败文案改成合约诊断，直接说明慢多少、缺哪个动作或是否破坏无失误，并给下一次路线动作。
- Drill HUD 增加 Clean/Pace/Style/Expert 合同条，玩家在跑动中能看到当前房间掌握阶梯。
- 质量门补上训练计划、死亡处方和合同条，保证训练闭环不会被后续视觉或地图改动冲掉。

## 2026-06-02 手感可信超大步

目标是让玩家相信输入窗口，而不是把成功归因于运气、把失败归因于系统。

- 跳跃 apex 加入轻量重力塑形，跳顶更容易微调，不改变核心速度上限。
- 命中 coyote、jump buffer、wall grace、Spark 压线或冲刺方向记忆时，会在角色旁给出短反馈。
- 输入提示新增 apex 稳定线，调试面板显示 feel cue、apex 和 aim memory 状态，便于后续继续调手感。
- 质量门补充 feel cue、apex gravity 和调试可见性，防止手感反馈被后续改动冲掉。

## 2026-06-02 路线清晰度超大步

目标是让玩家少猜“下一步该怎么走”，并在完成后立刻知道自己是不是更强了。

- 房间入场卡显示当前 SAFE/FAST/EXPERT 推荐线、路线动作和 mastery 分数。
- 奔跑中加入低打扰路线卡，短时间显示本房推荐路线、pace 状态和掌握阶段。
- 角色附近加入路线指南针，指向下一个关键实体或出口，Drill 中常驻，普通跑动中短暂出现。
- 清房和 Drill 成功后弹出 mastery 反馈，显示分数增量、clean/PB/合同完成和当前评级。
- Debug 和质量门补上 route cue、route compass 与 mastery popup，防止路线可读性被后续迭代冲掉。

## 2026-06-02 掌握路线图超大步

目标是把复盘从“看数据”推进到“马上知道下一步点什么”。

- 终点复盘新增掌握路线图，按房间缺口排序展示下一批 Clean/Pace/Style/Expert 合同。
- 路线图每一行都是 Drill 入口，点击后直接进入对应房间和合同，不需要再回设置面板查表。
- 设置掌握表加入 C/P/S/X 合同 pill，区分完成、尝试、当前推荐和未开练。
- Drill 完成反馈和 mastery toast 都会追加下一步合同，例如“下一 Pace Drill”或“维护 PB / Flow”。
- 质量门补上路线图 helper、合同 pill 和 CSS，保证训练闭环不会退回纯文字复盘。

## 2026-06-02 失败演练超大步

审查结论：当前死亡反馈已经能解释原因，但下一次尝试仍需要玩家把文字转换成路线动作。目标是把失败改成即时训练入口。

- 死亡、手动重开、房间重开都会保留失败演练状态，复活后不丢失修正目标。
- Drill 未达标会在自动重练时创建合同化演练提示，继续指向 Clean/Pace/Style/Expert 对应路线。
- 死亡点追加 REHEARSE 标记，场景内修正卡显示原因、下一动作、建议 Drill 和下一 mastery 合同。
- 玩家复活后会看到指向下一目标实体/出口的修正箭头，和路线指南针保持一致。
- 失败演练期间压缩重复屏幕卡片：死亡 Coach 只在死亡瞬间显示，复活后让场景演练承接信息，避免路线卡、入场卡和 Coach 同时抢画面。
- Debug 和质量门新增 failure rehearsal 检查，防止失败反馈退回只在角落显示文字。

## 2026-06-02 首屏与设置精修超大步

审查结论：训练系统已经厚，但首屏和设置仍偏原型；同时 canvas 提示文字被像素化缩放影响，导致提醒看起来发糊。目标是让第一眼更正式，设置更像训练驾驶舱。

- 启动层改为 ready 面板：显示训练档案状态、基础操作、继续奔跑和“先看训练”入口。
- “先看训练”可以在开跑前直接打开训练驾驶舱，玩家不用先进入游戏再找设置。
- 设置面板从窄单列改为桌面双列：左侧集中下一步、训练计划和连续训练；右侧集中房间合同、画面控制和复盘。
- 启动层在设置打开时自动退到背景，避免巨大标题和设置面板互相抢画面。
- Canvas 去掉 pixelated 缩放，改用受限高 DPI 缓冲，提升游戏内提示字清晰度，同时把上限控制在 1.25x 保持性能和截图稳定。
- 失败演练卡字体和尺寸上调，关键建议不再像压缩调试文本。
- 质量门新增 start panel、settings cockpit、canvas clarity 和训练入口检查，防止首屏精修回退。

## 2026-06-03 最终首房体验超大步

审查结论：首屏已比原来正式，但进入第一房后仍可能同时看到入场卡、路线卡、死亡 Coach 和失败演练，第一次玩的用户会把“提示很多”误读成“游戏很乱”。本轮目标是让前 3 分钟更清楚，而不是继续堆信息。

- 新增 `gameTip` DOM 提示条，字体和对比度独立于 canvas 缩放，解决小字发糊和截图不精致的问题。
- 首房未清且没有 R1 PB 时触发微引导，按移动、落点、冲刺、尖刺读法分段出现；不会影响已有存档玩家。
- 前几次死亡只显示一句下一次动作建议，例如先停安全格、晚一点花冲刺、脆冰只借一步，避免新手被完整训练系统压住。
- 入场卡、路线卡和死亡 Coach 在高优先级新手提示出现时让位；失败演练仍保留场景标记和路线箭头，保持训练价值。
- Debug 与质量门补充提示条状态、DOM id、样式和微引导 helper，避免后续 UI 调整把新手体验删回去。

## 2026-06-03 线上装入可靠性补强

审查结论：本地体验通过后，线上仍可能因为 GitHub Pages 和浏览器缓存短时间拿到旧 JS/CSS。对玩家来说，这不是“部署延迟”，而是打开页面后体验不一致。

- HTML 中的 CSS/JS 引用改为带版本号的资源地址，推送后能主动打破返回玩家的旧资源缓存。
- HTML 增加 build version 标记，线上排查时能快速确认玩家页面是否拿到新构建。
- 启动面板新增延迟显示的 `bootFallback`：如果 JS 没有成功初始化，玩家会看到刷新/缓存提示，而不是只面对一个不会响应的开始按钮。
- 启动面板新增 `noscript` 提示：禁用脚本时直接说明需要启用 JavaScript。
- JS 初始化完成后写入 `app-ready`，正常玩家不会看到兜底提示。
- 竖屏/侧边栏短视口下启动层脱离 16:9 舞台高度，使用全视口 overlay 和压缩 start panel，避免标题或按钮被裁掉；点击“先看训练”后 overlay 降层并禁用指针，设置面板保持可操作。
- 质量门检查资源版本号、build version、启动兜底样式、noscript 和 JS ready 标记，保证发布链路体验不再依赖人工记忆。

## 2026-06-03 控制可达性补强

参考《蔚蓝》这类成熟平台跳跃的经验，高手上限不应该只靠手指疲劳来维持。抓墙是高频动作，长期练习时持续按住会增加负担。

- 设置面板新增“抓墙：按住 / 切换”，默认仍为按住，保留原始手感。
- 切换模式下每次按抓墙键会保持或松开抓墙，键盘、触控和手柄输入共用同一状态。
- 出生、重开、死亡、切换键位或关闭输入时都会清空 latch，避免玩家误以为角色被卡住。
- Debug 会显示当前抓墙模式和 latched 状态，方便后续继续调输入可达性。
- 质量门新增抓墙模式 DOM、设置默认值和输入 helper 检查。

## 2026-06-03 P2 长期体验与方向守则

审查结论：前期和训练体验已经明显增强，但通关后仍需要更明确的长期目标；同时后续大步推进需要统一分支、PR、数据和 UI 规则，避免功能越加越散。

- 新增 `DEVELOPMENT_DIRECTION.md`，明确大步改造走 `codex/` 分支和 PR，不强推 main；P2 必须复用真实 PB/Clean/S/Style/Expert/Flow 数据。
- 设置面板新增章节完成度，综合显示通关房、Clean、S、Style、Expert 和章节等级。

## 2026-06-03 综合推进包

审查结论：训练系统已经厚，下一步必须同时抬高本体反馈、动作语言和可验证性，不能只继续堆面板。

- 新增 WebAudio 短音色：跳、墙跳、冲刺、Spark、光继、棱镜、回声、召回、弹簧、落地、死亡和清房都有输入确认。

- 失败演练新增 ghost 线：死亡或 Drill 失败会冻结最近动作轨迹，复活后把文字建议落回场景动作。
- 设置面板新增三步航线合同：稳定航线、节奏航线、高手航线会自动推进下一 Drill，让训练从单房变成连续课表。
- 背景新增章节共鸣：Flow、长期挑战或航线合同激活时，天空出现低透明状态线，服务作品感但不遮挡危险物。
- 质量门新增 feel replay fixtures、落脚平台数量检查和航线合同检查，后续调手感或地图不会只靠主观感觉。

## 2026-06-03 手感校准与航线闭环

审查结论：有了新动作和新合同后，最重要的是让玩家能从训练驾驶舱直接验证手感，并让连续训练的状态在跑动、复盘和中断时都说清楚。

- 设置面板新增 Feel Lab，按当前合约进度排序展示 jump buffer、coyote、wall grace、aim memory、Spark、Wall Spark 和 Prism Spark 校准卡，点击后直接进入对应 Drill。
- 航线合同进入 Active Drill HUD、练习报告和登顶复盘；完成时记录结果，中途改练、跳房或重开会保留“停在哪一步”的中断说明。
- 声音设置新增试听按钮，玩家不需要进房试错就能确认 WebAudio 是否可用、音量是否合适。
- 质量门继续扩展：新 DOM id、Feel Lab helper、航线 HUD/review helper、航线中断 helper、试听 helper 和样式都进入 `npm run check`。
- HTML 资源版本在该轮推进，避免本轮 JS/CSS 更新被旧缓存遮住。

## 2026-06-03 可用性自查固化

审查结论：上一轮依靠人工 HTTP smoke 已经能兜住 Browser 工具不可用的问题，但这不应靠记忆重复执行。本轮把它变成工程质量门。

- 新增 `npm run smoke`，脚本会自动找空闲端口、启动 `game-server.js`、请求首页、CSS 和 JS，并检查 build version、训练入口、Feel Lab、航线合同、试听按钮和 app ready helper。
- `npm run check` 现在包含 smoke，避免“脚本检查通过但本地服务资源 404/旧版本”的发布回归。
- Feel Lab 卡片新增进行中、刚完成和已中断状态；成功 Drill 会把本次校准用时写回卡片，切换训练或跳房会保留中断原因。
- 移动端下航线合同和 Feel Lab 卡片缩短高度，Feel Lab 变成单列，减少设置面板窄屏挤压。
- 质量门从硬编码 p5 改为 build version 一致性检查，后续升级 pN 只需改 HTML 资源版本。

## 2026-06-04 浏览器级与状态机加固

审查结论：HTTP smoke 能防资源 404，但不能证明玩家真的能点到训练入口，也不能覆盖状态串台。结合独立审查反馈，本轮优先修真实可用性。

- 新增 `npm run browser-smoke`，通过本机 Chrome/Edge DevTools Protocol 执行桌面 1280x720 和移动 390x700 检查，覆盖开始按钮、设置打开/关闭、声音试听、Feel Lab 点击、航线合同点击、坏 localStorage 恢复和触控 UI。
- Route 合同自动推进加入 generation 和 timer 清理，旧的 80ms 回调不能推进新合同；切换合同会记录中断，卡片可继续对应步骤。
- Flow 长期挑战改为本轮 Flow 达标，历史 Best 只作为提示，避免一开跑就被旧成绩判定达成。
- Drill 失败重练前会校验 Route/Feel 状态是否匹配当前 room/mode，不匹配则取消，避免错误状态残留。
- 存档读取统一有限数值、严格布尔、坏 JSON 自愈和规范化写回；长期档案会显示轻量 storage 状态，不打断游玩。
- 移动端设置面板使用 `100dvh` 和 safe area，Route/Feel/Review 文本使用安全断行，减少窄屏裁切和长 token 溢出。

## 2026-06-04 P10 可用性循环审查

审查结论：状态机和 smoke 已经可用，但真实可用性还缺“看得见、动得了、能继续、能自愈”的硬断言。本轮把这些薄弱面收进质量门。

- 浏览器 smoke 增加 canvas 像素非空、键盘位移和手柄 mock；手柄死区变成设置项，默认 0.28，可在 0.12-0.45 间调节。
- 坏存档自愈会显示一次性 toast，并继续在长期档案里保留轻量 storage 状态。
- Route 合同中断卡新增“继续上次”标签，降低玩家从训练驾驶舱恢复课表时的认知成本。
- Drill/Route/Challenge/Feel 的清理入口收进 `TRAINING_TRANSITIONS`，后续新增训练状态先过小型 transition 表。
- 终点复盘卡分为 primary/secondary，移动端优先读下一 Drill、章节、挑战和航线结果。
- Wall Spark / Prism Spark 触发时叠加更长的 variant pulse；失败 ghost 线增加方向箭头，让修正路线不只是一条静态线。
- 新增 `npm run route-audit`，检查十房路线文案、可读落点、航线合同、Feel Lab fixtures、transition guard 和关键视觉 helper；`npm run check` 已包含该门。

## 2026-06-04 P11 可用性与状态矩阵

审查结论：p10 已经兜住可点击、可读和可恢复，但启动页仍多一步，状态语义还缺专门矩阵，低性能和触控舒适设置也没有进入存档 schema。

- 启动页新增直接继续推荐 Drill 的按钮；有训练数据时显示“继续 Rn Drill”，点击后清掉旧复盘摘要并进入推荐房间合同。
- 设置新增低性能模式和触控大小；低性能只降低粒子、星点、背景飘带和 wake 数量，不改变危险物、路线提示或判定。
- Settings、Profile、Room Focus 写入 schema version；Room Focus 兼容旧数组，并迁移到 `{ schemaVersion, rooms }` 包装。
- 新增 `npm run state-check`，检查 hard reset / jump room transition 表、Route/Feel 保留条件、失败重练校验、直接继续入口和 schema 版本。
- Browser smoke 增加直接继续、schema 迁移、移动横屏设置面板和终点复盘滚动安全检查。

## 2026-06-05 P12 验证与人工测试闭环

审查结论：p11 的功能面已经足够厚，继续推进最有价值的是让真实使用路径更难回退，并明确哪些结果仍需要真人或实机验证。

- Browser smoke 增加键盘 `O/Escape` 设置入口，验证无鼠标路径不会失效。
- Browser smoke 增加 Route 合同中断、训练驾驶舱“继续上次”回显和恢复启动，覆盖连续训练断点。
- Browser smoke 增加 Feel Lab 改练中断回显，防止校准状态和普通 Drill 串台。
- 新增 `PLAYTEST_CHECKLIST.md`，把首三分钟、十房路线、Route/Feel、移动端和舒适设置变成可重复人工验收。
- 新增 `KNOWN_ISSUES.md`，把实机手柄、真实触控、完整真人十房、线上 Pages freshness 和音频感知列为当前仍需确认的边界。
- Contract check 要求 README/发布清单引用人工试玩与已知限制，避免后续只看脚本通过就误判成公开版本完全稳态。

## 2026-06-05 P13 反馈诊断闭环

审查结论：有了人工清单后，还需要一个低摩擦、无遥测的复现摘要，否则真人反馈容易停留在“感觉不对”。

- 设置面板复盘区新增“诊断 / 复制”，生成本地 JSON 摘要。
- 摘要包含 build、设置、视口、存档存在性、当前 run、Route/Feel/Challenge 和章节进度。
- 摘要明确排除身份信息、user agent、原始输入历史、replay path 和 secrets。
- Clipboard 不可用时回退为本地 JSON 下载，不上传、不改分数、不写训练状态。
- Browser smoke 会点击诊断按钮并检查快照结构；HTTP smoke 和 contract check 会守住入口和 helper。

## 2026-06-05 P14 反馈备注与输入隔离

审查结论：只有诊断摘要还不够，真人反馈需要一句“哪里不对”；但文本输入不能被全局热键污染。

- 设置面板复盘区新增反馈类型和 240 字反馈备注。
- 诊断快照会带上本次反馈类型和清洗后的备注，不写入 localStorage。
- 备注框内输入 `O` 不再触发设置开关；`Escape` 仍然关闭设置。
- Browser smoke 覆盖备注输入隔离和诊断备注入包，防止反馈面板以后变成“能看不能写”。

## 2026-06-05 P15 存档迁移与视觉门

审查结论：人工反馈闭环已经能描述问题，下一步要降低“复现状态”和“迁移存档”的成本，同时把移动端可读性变成浏览器门。

- 设置面板新增反馈模板复制，把版本、视口、训练状态、手柄摘要和复现空位整理成可提交文本。
- 本地存档新增复制、下载和导入 `summit-spark-save` JSON；导入走现有 normalize/schema 逻辑并刷新页面。
- 设置面板显示非敏感手柄状态，只报告连接数量、standard mapping 和当前动作摘要，不记录手柄 ID。
- Browser smoke 新增存档导出/导入、反馈模板和 360px 移动端视觉溢出检查。
- 诊断快照新增手柄摘要，仍不包含 user agent、身份信息、原始输入历史或 replay path。

## 2026-06-05 P16 导入预检与失败保护

审查结论：有导入能力后，最大风险是玩家粘错 JSON 后不知道发生了什么，甚至误以为存档被覆盖。本轮把导入前的可读反馈和失败路径补进产品与浏览器门。

- 存档导入框新增实时预检状态，显示来源版本、登顶数、房间 PB、Flow 和触控大小。
- 无效 JSON、错误 kind 和过大内容会原地提示错误，不刷新页面、不写 localStorage。
- 有效导入继续走同一套 normalize helper，预检和实际写入不会分叉。
- Browser smoke 覆盖无效 JSON、错误 kind、有效预览、导入刷新和 schema 写回。

## 2026-06-05 P17 体验降噪与设置重排

审查结论：功能已经堆得足够多，当前玩家第一感受的问题是“太蓝、太吵、说明太多、设置太乱”。本轮优先降噪，而不是再加玩法。

- 启动页文案收缩为一句短目标，基础操作变成四个短标签。
- 首房自动提示从四段教学减到两段短提示，降低平台跳跃游戏不必要的解释负担。
- 设置面板改成折叠分组：训练和房间默认打开，控制、反馈、档案按需展开。
- 配色从蓝色发光改为石墨/暖灰基底，只保留金色和绿色作为重点状态。
- Browser smoke 适配折叠分组，先展开目标分组再验证试听、反馈、存档、Route 和 Feel 路径。

## 2026-06-05 P18 极简首屏与字体修正

审查结论：p17 仍然残留“说明页”和重字重面板感。当前更值得做的是减少默认展示，而不是继续解释玩法。

- 启动页删除目标说明和操作指导，只保留标题、开始、继续和设置入口。
- 自动新手提示、死亡教学 toast 和普通训练启动说明静默，必要错误、存档和诊断反馈保留。
- 设置面板默认全折叠，打开时先像系统设置目录，而不是直接铺满训练卡和反馈表单。
- 字体改为系统字体栈，UI 和 canvas HUD 从 800/900 重字重降到更轻的层级。
- Canvas 房间说明从大教学卡压成房间/target 小状态 chip，并且未开始时不再显示。
- Smoke/contract/browser smoke 改为守住“无首屏说明块”和“设置默认全收起”的真实 UX。

## 2026-06-05 P19 纯体验模式与设置面板再整理

审查结论：p18 已经去掉首屏说明，但游戏内仍残留死亡修正、失败演练、首次输入计时提示等训练语。玩家要“好好体验游戏”时，这些信息会破坏节奏。本轮把训练能力留在设置/Drill 里，默认跑动只保留必要反馈。

- 正常游玩不再显示 fall/死亡修正、失败演练卡、死亡点文字、死亡轨迹回放和首次输入计时提示。
- 死亡、手动重开、房间重开和 Drill 未达标仍保留统计、状态清理和快速重试，不再把下一动作处方压到画面上。
- 设置面板重排为单列系统设置：控制默认展开，声音、显示、房间、反馈与存档、档案、训练、进阶按需打开。
- 声音试听从控制组拆出，显示/性能/练习线/影子/震动/调试统一放进显示组，避免一个分组塞满所有杂项。
- Smoke、contract、route audit 和 Feel replay 检查改为守住静默体验，防止“修正路线 / 首次输入开始计时”等旧提示回流。
